# Jolt Time — System Documentation

## Project Identity

**Project Name:** Jolt Time (Струс Часу)
**Type:** Telegram Mini App — Educational Entertainment Game
**Core Concept:** Time-travel themed game where players explore historical civilizations
**Platform:** Telegram Mini App (mobile-first)
**Revenue Model:** Free-to-play with AdsGram advertisements as primary revenue source

## Technical Stack

- **Frontend:** Web-based Telegram Mini App (HTML/CSS/JavaScript or React)
- **Backend:** Cloud-based API (Node.js/TypeScript recommended)
- **Database:** Supabase (PostgreSQL + Auth + Realtime)
- **Notifications:** Telegram Bot API
- **Version Control:** GitHub

## Architecture Principles

1. **Scalability** — Architecture supports large player base and future multiplayer
2. **Security** — User data protection, secure authentication, input validation
3. **Performance** — Optimized for mobile devices, fast load times
4. **Maintainability** — Clean code structure, comprehensive documentation
5. **Extensibility** — Modular design for future features

## Quality Standards

- Premium quality over quick releases
- Educational value in all content
- Accessible to casual players
- Consistent user experience
- Regular updates and improvements

## Communication Channels

- GitHub Repository for source code and documentation
- Telegram Bot for player notifications
- In-app feedback system

## Development Guidelines

- Mobile-first design approach
- Progressive Web App capabilities
- Offline-first with sync capabilities
- A/B testing infrastructure
- Analytics integration
- Error tracking and monitoring

## Important Disclaimers

- **NOT a Play-To-Earn game**
- Players never earn real money
- AdsGram advertisements are PRIMARY revenue source
- Educational entertainment focus
- Long-term ecosystem development
