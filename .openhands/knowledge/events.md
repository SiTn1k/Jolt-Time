# Jolt Time — Events System

## Overview

Events provide limited-time content that keeps the game fresh and exciting while strengthening Jolt Time's educational identity through historical celebrations. All events are cosmetic-focused with no pay-to-win rewards. Events create opportunities for all players to earn unique rewards regardless of spending, encourage daily engagement without creating obligation, and deliver fresh experiences that respect player time.

---

| Event | Duration | Description |
|-------|----------|-------------|

| Collection Marathon | 7 days | Extended event focusing on artifact gathering |

### Seasonal Events

Four major events per year, each 14 days:
- **Spring:** Blossom Festival (March-April)
- **Summer:** Solstice Celebration (June-July)
- **Autumn:** Harvest Moon (September-October)
- **Winter:** Winter Wonderland (December-January)

Real-world holiday celebrations with themed content:
- New Year Celebration (December 31 - January 5)
- Valentine's Day (February 14-16)
- Easter (March-April, varies)
- Halloween (October 25-31)
- Christmas (December 20-26)

### Community Events

Collective goals that unite all players:
- **Frequency:** 2-3 times per season
- **Duration:** 3-7 days
- **Requirements:** Community-wide participation unlocks rewards for all

### Anniversary Events

Annual celebrations marking Jolt Time's launch:
- **Frequency:** 1 per year
- **Duration:** 30 days
- **Special:** Major content reveals, founding player recognition

## 11. Event Philosophy

Jolt Time's event system embodies principles of fair celebration, educational value, and respectful engagement.

### Core Principles

#### Provide Fresh Content Regularly
- New events every 2-3 weeks
- Rotation of event types prevents monotony
- Seasonal themes create anticipation
- Historical events reinforce educational mission

#### Reward Participation
- All participants earn something
- Multiple reward tiers for different engagement levels
- Casual players can complete events
- Dedicated players get exclusive recognition

#### Avoid Excessive Grinding
- Clear time estimates for rewards
- No "must play X hours per day" requirements
- Daily caps prevent obsession
- Events completable at comfortable pace

#### Respect Player Time
- No mandatory daily login
- Missed days don't ruin event progress
- Flexible participation windows
- Offline progress continues

### Anti-Addiction Measures

| Concern | Solution |
|---------|----------|
| FOMO Pressure | Events repeat yearly, no permanent content missed |
| Grind Pressure | Clear completion thresholds, no endless farming |
| Social Pressure | Optional leaderboard visibility |
| Loss Aversion | Tokens convert to dust if unspent |
| Time Pressure | Events are celebrations, not obligations |

### Casual Player Support

| Player Type | Event Accessibility |
|-------------|---------------------|
| Casual (5 min/day) | Complete easy missions, earn participation rewards |
| Regular (15-30 min/day) | Complete most missions, earn common/uncommon rewards |
| Dedicated (60+ min/day) | Complete all missions, compete for rare rewards |
| Completionist | Pursue leaderboard positions, collect everything |

### Event Balance Checklist

- [ ] No gameplay advantages for spenders
- [ ] Clear reward expectations displayed
- [ ] Achievable by casual players (5-10 hours total)
- [ ] Fun and engaging activities
- [ ] Beautiful cosmetic rewards
- [ ] Respects player time (no mandatory daily login)
- [ ] Proper notification controls
- [ ] Server capacity tested
- [ ] Educational content integrated naturally
- [ ] Fair leaderboard distribution

## 15. Event Technical Design

### Event Data Structure

```yaml
event:
  id: "egypt_week_2026"
  name: "Ancient Egypt Week"
  type: "historical"
  era: "egypt"
  
  timing:
    start: "2026-06-15T00:00:00Z"
    end: "2026-06-22T00:00:00Z"
    duration_days: 7
    
  content:
    missions: 10
    cosmetic_rewards: 5
    community_goal: true
    token_reward: 2000
    
  participation:
    tracked: true
    rewards_claimable: true
    leaderboard_active: true
    
  localization:
    available_languages: ["en", "es", "fr", "de", "ja", "zh", "ar"]
    
  adsgram:
    bonus_available: true
    daily_limit: 3
```

```yaml
event_currency:
  token:
```

## 16. Quality Standards

### Event Pre-Launch Checklist

- [ ] All missions tested and balanced
- [ ] Reward acquisition rates verified
- [ ] Leaderboard distribution analyzed
- [ ] Notification timing confirmed
- [ ] Localization complete
- [ ] Server load testing performed
- [ ] Fallback systems in place
- [ ] Community guidelines ready
- [ ] Social media assets prepared
- [ ] Support team briefed

### Event Post-Launch Checklist

- [ ] Monitor participation rates
- [ ] Track reward distribution
- [ ] Watch for exploits
- [ ] Gather player feedback
- [ ] Document lessons learned
- [ ] Archive event for reference
- [ ] Plan improvements for next year

---

*Events should feel like celebrations, not obligations. Every player should find joy, not pressure.*
