# Historical Content Pipeline

## Overview

This document defines the complete workflow for creating, reviewing, and maintaining historical content in Jolt Time. Historical content is central to the project's identity—accuracy and educational value always take priority over content quantity.

**Content Focus:** Historical accuracy, educational value, player engagement
**Core Principle:** Respect history, avoid misinformation, inspire curiosity

---

## Content Categories

Historical content spans five core categories:

| Category | Purpose | Priority |
|----------|---------|----------|
| **Historical Artifacts** | Real artifacts from history with accurate descriptions | Highest |
| **Historical Events** | Real events presented in engaging ways | Highest |
| **Educational Articles** | In-depth historical context and connections | Medium |
| **Character Stories** | Fictional characters within historical settings | Medium |
| **Museum Descriptions** | Concise summaries for collected items | High |

---

## Historical Philosophy

### Core Principles

1. **Respect History**
   - Real people, events, and artifacts receive accurate treatment
   - Cultural sensitivities acknowledged and respected
   - No manipulation of historical facts for gameplay convenience

2. **Avoid Misinformation**
   - Disputed facts clearly labeled
   - Corrections addressed promptly
   - No presenting speculation as fact

3. **Prioritize Educational Value**
   - Content encourages genuine curiosity
   - Players learn something meaningful
   - Links to further learning provided

### What This Means

- **Accuracy over Speed:** New eras release only when content is verified
- **Quality over Quantity:** Fewer well-crafted artifacts beats many shallow ones
- **Learning over Gating:** Historical context enhances rather than blocks progress

## Content Creation Workflow

### Stage Overview

```
Research → Drafting → Review → Publication → Maintenance
```

### Stage 1: Research

**Activities:**
- Source identification and verification
- Historical context gathering
- Cultural sensitivity review
- Academic consultation (where needed)

**Standards:**
- Primary sources preferred over secondary
- Peer-reviewed sources required for controversial topics
- Diversity of perspectives represented

**Deliverables:**
- Research brief document
- Source bibliography
- Cultural sensitivity notes

### Stage 2: Drafting

**Activities:**
- Initial content creation
- Lore integration
- Gameplay context alignment
- Accessibility formatting

**Standards:**
- Age-appropriate language (target: 13+)
- Clear, engaging prose
- Consistent terminology
- Visual asset specifications

**Deliverables:**
- Content draft
- Image/asset references
- Integration notes

### Stage 3: Review

**Review Layers:**

| Review Type | Focus | Reviewers |
|-------------|-------|-----------|
| **Historical Review** | Factual accuracy, source quality | Historical consultants |
| **Editorial Review** | Writing quality, consistency | Content editors |
| **Cultural Review** | Sensitivity, representation | Cultural consultants |
| **Gameplay Review** | Integration, engagement | Game designers |

**Standards:**
- Two independent reviewers per content piece
- All flags must be resolved before approval
- Documentation of all changes

**Deliverables:**
- Approved content
- Review comments (resolved)
- Sign-off documentation

### Stage 4: Publication

**Activities:**
- Final QA pass
- Localization preparation
- Asset integration
- Content management system entry

**Standards:**
- No breaking changes after publication
- Content versioning tracked
- Rollback capability maintained

**Deliverables:**
- Published content
- Version records
- Integration confirmation

### Stage 5: Maintenance

**Activities:**
- Accuracy monitoring
- Player feedback integration
- Periodic review cycles
- Correction protocols

**Standards:**
- Player-submitted corrections reviewed within 30 days
- Annual accuracy audit of all published content
- Version history preserved

## Source Philosophy

### Source Standards

| Source Type | Acceptability | Examples |
|-------------|---------------|----------|
| **Primary** | Preferred | Museum catalogs, academic journals |
| **Secondary** | Acceptable | History books, documentaries |
| **Tertiary** | Limited | Wikipedia (for overview only) |
| **Unknown** | Rejected | Unsourced claims, rumors |

### Source Requirements

- **Credibility:** Sources must have clear authorship and publication
- **Recency:** Information should reflect current scholarly consensus
- **Diversity:** Multiple perspectives represented where applicable
- **Transparency:** Sources documented and available for review

### What We Avoid

- **Speculation as Fact:** Clearly label theoretical content
- **Single-Source Claims:** Major facts require multiple sources
- **Outdated Information:** Update when new research contradicts old
- **Cultural Appropriation:** Respect source cultures

**Standards:**
- Style guide adherence
- Consistent terminology

- Grammar and spelling

| Check | Frequency | Responsible |
|-------|-----------|-------------|
| Factual verification | Per content | Historical reviewer |
| Style compliance | Per content | Editorial reviewer |
| Accessibility audit | Per content | QA team |
| Cultural sensitivity | Per era | Cultural advisor |
| Integration test | Pre-publication | Dev team |

## Localization Notes

### Localization Principles

1. **Understandable**
   - Simple language that translates well
   - Avoid idioms and cultural references
   - Clear sentence structure

2. **Culturally Respectful**
   - Localizations reviewed by native speakers
   - Cultural sensitivities addressed per region
   - Historical context preserved

3. **Consistent**
   - Same terminology across translations
   - Consistent historical names and dates
   - Unified formatting

### Translation Workflow

- Initial translation by professional translators
- Review by native speakers with historical knowledge
- Gameplay integration testing
- Ongoing correction as needed

## Educational Philosophy

### Core Approach

1. **Encourage Curiosity**
   - Open-ended questions in content
   - Links to further resources
   - Discovery moments in gameplay

2. **Reward Learning**
   - Knowledge recognized in achievements
   - Quiz content for deeper engagement
   - Collection incentives for educational completion

3. **Remain Accessible**
   - No academic jargon without explanation
   - Multiple depth levels (summary + detail)
   - Visual learning aids where possible

### Educational Integration

- **Artifacts:** Historical context on collection
- **Events:** In-game historical mini-lessons
- **Museum:** Organized educational displays
- **Achievements:** Learning milestones recognized

## Museum Content Standards

### Description Guidelines

| Element | Standard |
|---------|----------|
| **Length** | 50-150 words |
| **Structure** | Hook → Context → Significance |
| **Tone** | Engaging but informative |
| **Reading Level** | 13+ |

### Content Structure

**Opening (1-2 sentences):**
- What is it?
- When/where is it from?

**Context (2-3 sentences):**
- Historical significance
- Why it matters

**Connection (1-2 sentences):**
- Relevance to player
- Collection significance

### Quality Checklist

- [ ] Factual accuracy verified
- [ ] Cultural sensitivity reviewed
- [ ] Length within guidelines
- [ ] Appropriate for all ages
- [ ] Consistent with existing content
- [ ] Links to further resources included

## AdsGram Notes

### Monetization Boundaries

Historical content remains uninterrupted by aggressive monetization:

- **No Interruption:** Educational content never blocked by ads
- **Optional Viewing:** Ad-watched content supplements, never gates
- **Clear Rewards:** Reward value shown before watching
- **Respectful Frequency:** No repeated prompts for same content

### Content Protection

- Historical accuracy never compromised for monetization
- Educational value maintained regardless of ad integration
- Player trust prioritized over short-term revenue
