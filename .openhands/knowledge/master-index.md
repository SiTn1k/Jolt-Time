# Jolt Time — Master Index

## Welcome to Jolt Time

This is the official entry point for understanding the entire Jolt Time ecosystem. Jolt Time is a Telegram Mini App + Telegram Bot game built on Supabase, where players travel through time to discover historical anomalies, collect artifacts, and restore the timeline.

**Quick Start:** Browse this index to find any topic, or start with [Project Vision](project-vision.md) to understand our mission and principles.

---

## Project Summary

### Mission
Jolt Time exists to make history exciting, accessible, and fun for everyone. Learning about the past should feel like an adventure, not a chore.

### Vision
Build a premium educational entertainment experience that sparks curiosity about history while providing engaging gameplay and sustainable monetization.

### Core Principles
- **Accessibility** — Everyone deserves to play
- **Fairness** — No pay-to-win mechanics
- **Educational Value** — Accurate history, inspired curiosity
- **Long-Term Sustainability** — Build to last
- **Respect for Players** — Players are partners, not products

**See:** [Project Vision](project-vision.md) for complete vision and principles.

## Architecture Overview

### Frontend (Telegram Mini App)

**Technology:** TypeScript, React, CSS

**Key Components:**
- Mini App shell with Telegram WebApp SDK
- Screen-based navigation
- State management (Zustand/Redux)
- Component library with design system
- Service worker for offline support

**See:** [Screens](screens.md), [UI Style](ui-style.md), [Navigation](navigation.md)

### Backend (Supabase)

**Technology:** Supabase (PostgreSQL, Edge Functions, Auth)

**Key Components:**
- PostgreSQL database with RLS
- Edge Functions for API logic
- Auth for authentication
- Realtime for live updates
- Storage for assets

**See:** [Database Schema](database-schema.md), [API Architecture](api-architecture.md)

### Telegram Bot

**Technology:** Node.js, Telegram Bot API

**Key Components:**
- Command handlers
- Inline query handlers
- Callback query handlers
- Scheduled jobs
- Notification delivery

**See:** [Telegram Architecture](telegram-architecture.md), [Telegram Bot System](telegram-bot-system.md)

- Every document has a clear purpose
- Documents grouped by category
- Cross-references connect related content

- Update this index when adding new docs
