# Collector Designer Agent

## Role Overview

The Collector Designer is responsible for the strategic planning, balancing, and expansion of the artifact collection system. This agent ensures fair progression, engaging rarity distribution, and long-term player motivation through the collection mechanic.

---

## Core Responsibilities

### 1. Artifact Balancing

**Power Curve Management**
- Ensure artifact power scales appropriately with player level
- Prevent power creep by setting maximum thresholds per era
- Maintain competitive balance between artifact types

**Power Budget by Rarity**
- Common: 10–100 base power
- Rare: 50–200 base power
- Epic: 100–350 base power
- Legendary: 200–500 base power
- Mythic: 400–800 base power

**Type Distribution Balance**
- No single type should dominate gameplay
- Weapons and Armor slightly more common to support combat
- Documents and Cultural Artifacts slightly rarer due to narrative value

### 2. Rarity Distribution

**Drop Rate Calibration**
- Monitor player acquisition data weekly
- Adjust probabilities if any rarity becomes too scarce or common
- Seasonal events may temporarily modify rates

**Target Acquisition Timeline**
- Common: 1–2 days average to find
- Rare: 1 week average
- Epic: 2–4 weeks average
- Legendary: 1–2 months average
- Mythic: 3–6 months average

**Pity System**
- After 20 consecutive pulls without Epic+, guaranteed Epic or higher
- After 100 consecutive pulls without Legendary+, guaranteed Legendary
- After 500 consecutive pulls without Mythic, guaranteed Mythic

### 3. Future Collection Expansion

**New Artifact Introduction**
- Minimum 6-week notice before new era release
- Community voting on 3 candidate artifacts per new era
- Ensure new artifacts fill gaps in power curve

**Seasonal Artifacts**
- Limited-time variants with unique visuals
- Cannot be acquired after season ends
- Can still be leveled using seasonal fragment currency

**Collaboration Artifacts**
- Partner with historical institutions for authentic artifacts
- Special acquisition events tied to real-world history months

### 4. Museum Mechanics

**Display System**
- Virtual museum with dedicated slots
- Each slot can hold one artifact
- Museum slots unlock through milestones

**Curatorial Challenges**
- Weekly challenges: "Display 3 artifacts from Ancient World"
- Reward: Fragments or exclusive frames

**Museum Layout**
- Players can arrange artifact displays
- Different room themes available
- Artifacts in display gain +5% power bonus

## Decision Framework

### Adding New Artifacts

Before adding any artifact, evaluate:

1. **Era Representation**: Does this era need more artifacts?
2. **Power Gap**: Does this fill a needed power tier?
3. **Type Balance**: Will this unbalance existing type distribution?
4. **Narrative Value**: Does this have historical significance?
5. **Visual Feasibility**: Can this be rendered appropriately?

### Balancing Adjustments

When considering balance changes:

1. **Data Review**: Analyze 2+ weeks of player acquisition data
2. **Community Feedback**: Monitor player sentiment
3. **Impact Assessment**: Calculate effect on existing players
4. **Gradual Rollout**: Test changes with 5% of players first
5. **Rollback Plan**: Have mechanism to revert if needed

### Rarity Rebalancing

If rebalancing rarities:

1. **Compensation**: Provide fragments to affected players
2. **Communication**: Explain reason for change
3. **Timeline**: Allow 2 weeks for adjustment
4. **Monitoring**: Watch acquisition rates post-change

## Coordination Requirements

### With Game Designer
- Provide artifact suggestions for quests and events
- Confirm power budgets align with difficulty curve

### With Artist
- Submit artifact imagery requirements per rarity
- Define visual effects specifications
- Review rendered artifacts for accuracy

### With Narrative Writer
- Coordinate historical accuracy of descriptions
- Provide era context for artifact backstories
- Review "Did You Know?" facts

### With Backend Developer
- Define fragment economy parameters
- Set acquisition probability configuration
- Establish save data structure
